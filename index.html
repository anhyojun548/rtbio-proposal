업무를 더 스마트하게,
맞춤 자동화 시스템

엑셀과 수기 작업에서 벗어나, 클릭 한 번으로 발주부터 정산까지 자동으로 처리되는 시스템을 소개합니다.

현재 상황

지금 이런 불편함을 겪고 계시지 않나요?

현재 RTBIO는 하루 약 30건의 발주를 엑셀과 수기로 처리하고 있습니다. 이 과정에서 여러 문제가 반복적으로 발생합니다.

발주서, 거래명세서, 정산표에 같은 데이터를 반복 입력합니다.

수기 입력 과정에서 금액 오류, 수량 착오가 발생합니다.

📊

미수금이 얼마인지, 어떤 발주가 처리 안 됐는지 바로 확인할 수 없습니다.

🕐

월말 마감에 시간 소모

각종 자료를 모아 수작업으로 마감 보고서를 만드는 데 많은 시간이 걸립니다.

해결 방법

시스템이 업무를 이렇게 바꿔드립니다

발주 접수부터 정산, 보고서 생성까지 — 하나의 시스템에서 자동으로 처리됩니다. 경영지원팀은 확인 버튼만 누르면 됩니다.

1

거래처가 발주를 넣습니다

대리점이나 병원 담당자가 온라인으로 주문을 접수합니다. 발주번호가 자동으로 생성됩니다.

2

경영지원팀이 확인합니다

품목, 수량, 배송비가 자동 계산되어 표시됩니다. 이상 없으면 확정 버튼 한 번이면 끝.

3

출고 후 거래명세서가 자동 발송됩니다

출고를 확인하면 PDF 거래명세서가 자동 생성되어 거래처에 이메일로 발송됩니다.

4

정산이 자동으로 관리됩니다

거래처마다 설정된 정산 주기(월말, 60일 등)에 따라 미수금이 자동 집계됩니다.

5

월말 보고서가 자동 생성됩니다

마감 시점에 거래처별 원장과 보고서가 자동으로 만들어집니다. 더 이상 수작업이 필요 없습니다.

누가 이 시스템을 사용하나요?

각 부서와 거래처 담당자가 자기 역할에 맞는 화면만 보고 사용합니다. 다른 부서의 데이터는 볼 수 없어 보안이 유지됩니다.

주요 기능

시스템이 처리해 주는 일들

1단계에서 바로 사용할 수 있는 핵심 기능들입니다. 모두 자동으로 처리되기 때문에 경영지원팀의 수작업이 크게 줄어듭니다.

📦

거래처 & 품목 관리

대리점/병원 정보, 결제 조건, 배송비 기준을 한곳에서 관리합니다. 품목의 사이즈·색상·가격도 등록하면 주문 시 자동 반영됩니다.

🔢

자동 발주번호 & 배송비 계산

발주가 들어오면 고유 번호가 자동 생성됩니다. 배송비도 품목과 거래처 조건에 따라 자동 계산됩니다.

📄

거래명세서 자동 생성 & 발송

출고 확인 후 PDF 거래명세서가 자동으로 만들어져 거래처에 이메일로 보내집니다.

💰

정산 & 미수금 자동 관리

거래처마다 다른 정산 주기(월말, 60일 등)를 설정하면 시스템이 알아서 집계합니다. 미수금 현황도 실시간으로 파악됩니다.

📊

대시보드 & 월 마감 보고서

신규 발주, 처리 대기, 미수금 등 주요 현황을 한눈에 볼 수 있습니다. 월말에는 보고서가 자동으로 생성됩니다.

⚙️

거래처별 예외 설정

거래처마다 할인율, 배송비 면제, 외상 한도, 정산 주기를 각각 다르게 설정할 수 있습니다.

🏷️

의료기기 고유식별자(UDI) 관리

제품별 UDI 코드를 등록하고 납품 이력과 연결하여 추적할 수 있습니다. 규제 대응에 필요한 데이터를 체계적으로 관리합니다.

처음부터 모든 걸 만들지 않습니다. 가장 급한 것부터 시작하고, 현장에서 쓰면서 점차 확장합니다.

영업사원이 직접 거래처별 매출, 미수금, 실적을 조회할 수 있게 됩니다.

선택 1 / 3 — 시스템 구축 방식

기존에 사용하시는 ERP와의 관계를 어떻게 할지 결정합니다.

권장

새로운 맞춤 시스템

기존 ERP와 별도로, RTBIO 업무에 100% 맞춰서 새로 만듭니다. 기존 데이터는 옮겨올 수 있습니다.

  • 우리 회사 업무 방식에 딱 맞게 설계
  • 하나의 시스템만 사용하니 관리가 간편
  • 앞으로 기능 추가가 자유로움
대안

기존 ERP에 연결

지금 쓰고 있는 ERP에 새 시스템을 덧붙여 연결합니다.

연동 개발 복잡도로 추가 비용 +500만원 이상 발생
  • 기존 ERP 구조에 맞춰야 해서 자유도 제한
  • 두 시스템 사이 데이터 충돌 관리 필요
  • 기존 ERP 업체에 계속 의존 (이중 비용 발생)
  • 향후 확장 시 추가 연동 비용 반복 발생

선택 2 / 3 — 거래처 발주 방식

대리점이나 병원 담당자가 어떤 방식으로 주문을 넣을지 결정합니다.

권장

전용 웹사이트에서 직접 발주

거래처 담당자가 전용 URL에 접속해서 품목을 선택하고 수량을 입력합니다. 주문이 바로 시스템에 들어옵니다.

  • 주문 데이터가 정확하게 자동 입력
  • 수기 오류가 원천 차단
  • 추가 비용 없음
대안

기존 방식(카톡/엑셀) + AI 자동 인식

거래처는 지금처럼 카카오톡이나 엑셀로 발주하고, AI가 자동으로 시스템에 입력합니다.

AI 모듈 추가 개발로 +300만원 이상 발생
  • AI 인식 정확도가 100%는 아님 (수동 확인 필수)
  • 매월 AI 사용에 따른 토큰 비용 지속 발생
  • 비정형 데이터 인식 오류 시 발주 누락 위험

선택 3 / 3 — 사용 환경

PC와 모바일 모두에서 사용할 수 있는 방식을 결정합니다.

권장

반응형 웹사이트

PC, 태블릿, 스마트폰 모두에서 인터넷 브라우저로 접속합니다. 앱 설치가 필요 없습니다.

  • 설치 없이 바로 사용 가능
  • PC와 모바일 모두 지원
  • 개발 비용이 낮고 업데이트가 즉시 반영
대안

모바일 앱 (iOS/Android)

앱스토어에서 설치하는 전용 앱입니다.

iOS/Android 별도 개발로 추가 비용 +500만원 이상 발생
  • iOS + Android 각각 별도 개발 비용 발생
  • 앱스토어 심사 기간 필요 (1~4주 소요)
  • 앱스토어 설치 필요 (거래처 거부감 가능)
  • 업데이트마다 재심사 필요 (즉시 반영 불가)
  • 앱스토어 연간 등록비 별도 발생
